## 🏗️ 架构分层

### 1. 数据层 (Data Layer)

**模块结构**:
```
app/data/
├── tushare_client.py    # Tushare 数据源
├── akshare_client.py    # AKShare 数据源
├── data_fetcher.py      # 数据协调器
├── health_check.py      # 健康检查
└── preprocessor.py      # 数据预处理
```

**优点**:
- ✅ 多数据源支持（Tushare + AKShare）
- ✅ 自动降级熔断机制
- ✅ 统一的数据模型（DailyQuote、StockInfo）
- ✅ 异步数据获取（asyncio）

**待改进**:
- 🔲 批量获取优化（避免频繁API调用）
- 🔲 本地数据缓存（减少网络请求）
- 🔲 数据版本管理（支持历史回测）

**关键问题**:
1. **Q1**: DataFetcher 是否需要添加批量获取接口？
   - 当前: 每次分析一只股票
   - 建议: 添加 `get_batch_quotes(stock_codes: List[str])`

2. **Q2**: 是否需要本地SQLite存储？
   - 当前: 完全依赖在线API
   - 建议: 添加本地缓存，减少API调用次数

**关键问题**:

## 📊 数据流架构

```
用户命令
    ↓
stock_analyzer.py (CLI)
    ↓
DataFetcher (协调器)
    ↓
    ├─→ TushareClient (主数据源) → 成功 ✓
    │       ↓ 失败
    └─→ AKShareClient (备用数据源) → 降级 ✓
    ↓
Preprocessor (数据预处理)
    ↓
SystemAnalyzer (分析引擎)
    ↓
    ├─→ TrendIndicators (趋势指标)
    ├─→ MomentumIndicators (动量指标)
    ├─→ VolatilityIndicators (波动指标)
    └─→ VolumeIndicators (成交量指标)
    ↓
ReportGenerator (报告生成)
    ↓
    ├─→ HTML Report (可视化)
    └─→ Markdown Report (结构化)
    ↓
本地文件存储
```

**关键决策点**:
1. **数据源降级策略**: Tushare → AKShare
2. **熔断器阈值**: 5次失败后熔断，30秒恢复
3. **缓存策略**: Redis 可选，默认内存缓存
